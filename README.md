# 🎵 Music Dimension Visualizer - 前端交互分析系统

### 音乐流派与声学特征高维交互可视化驾驶舱

> **项目定位**：基于 Spotify 经典音乐数据集（1960-2020）的多维声学与文化特征交互分析系统。
> **技术栈**：`React 18` + `Vite` + `ECharts 5`
> **数据源**：`public/data.json`（已进行离线预处理与聚类归档）

---

## 🧭 核心功能板块与技术实现

系统围绕**流派归属**、**声学特征**、**情感空间**与**词频文化**四大维度，实现了高度关联的双向联动交互：

### 1. 🌊 流派结构与声学特征透视仪 (`ToggleableChordRadar`)
* **双层旭日图 (Sunburst)**：展示多级流派的父子层级结构。节点亮度引入**流行度自适应比例因子**，低流行度流派节点自动降低亮度（最高淡化 `55%`），辅助视觉分级。
* **声学特征雷达联动**：当鼠标悬停或点击旭日图节点时，联动雷达图实时绘制该流派在 **可舞性、能量、声学度、情感效价 (Valence)、活跃度、器乐度、言语度** 7 个维度上的特征多边形。

### 2. 🌀 流派专属画像悬浮窗 (Draggable Context Modal)
* **开启方式**：点击旭日图的子流派节点触发。
* **数据构成**：
  * **高频歌词词云**：展示该流派歌曲中前 20 位高频词汇。
  * **历代流行度变迁折线图**：绘制流派从 1960 至 2020 年的流行度走向，并提供“本流派”、“大分类”以及“大盘均值”的三线对比，用于时间变迁上的趋势洞察。
* **交互机制**：
  * **阻尼拖动 (Draggable)**：鼠标按住小窗口头部可在页面内平移，支持随时叉掉关闭。

### 3. 🔤 歌词词频多维特征透视仪 (Word Feature Analyzer)
* **开启方式**：在“流派画像”小窗口中点击词云的任何一个歌词标签触发。
* **计算逻辑**：
  * **流派亲和度**：遍历系统包含的所有流派，统计该词在各流派中的出现频次，排序并提炼其主要关联流派。
  * **情感声学指纹**：提取各关联流派的 7 维声学特征，以词频为权重进行**加权平均**，计算出该词在整个音乐库中的虚拟声学特征向量。
* **可视化方案 (无重复图表)**：
  * **流派特征：南丁格尔玫瑰图 (Nightingale Rose Chart)**：花瓣的半径和面积双重编码该词在不同流派中的浓度分布。
  * **情感特征：极坐标单轴放射柱状图 (Polar Radial Bar Chart)**：以极坐标放射线条的长度表征加权后的 7 维声学特征强度。
* **交互机制**：支持窗口关闭与鼠标任意位置的拖动平移。

### 4. 🌌 KMeans 情感空间星云图 (`ScatterBrushChart` / Voronoi map)
* **二维情绪空间**：横轴表示**能量 (Energy)**，纵轴表示**愉悦度 (Valence)**。
* **Voronoi 背景网格剖分**：计算 5 个情感聚类中心（质心），在 50×50 的网格平面上按最近距离自动剖分，生成五种色彩的情感边界区域（狂热释放、轻松愉快、伤感静谧、阳光活力、迷幻张力）。
* **优先级排他去重算法**：当大流派（Parent）与旗下子细分类同时被勾选为对比系列时，大流派系列会自动剔除子流派的歌曲，防止因为散点物理坐标重合而在 ECharts 中发生系列遮挡遮盖 Bug。
* **极简降噪高亮联动**：鼠标悬停在顶部的流派卡片时，该流派的散点保持尺寸不变，而**其他非悬停流派的散点透明度瞬间降至 `0.08`（淡化为虚影）**。移开后通过 React 虚拟 DOM 秒级重绘恢复（`0.88`），无任何 ECharts Action 残留 Bug。
* **歌曲定位映射**：在右侧列表点击歌曲，星云图会精确捕捉该歌曲的坐标，以**红色扩散动画高亮包裹对应节点**。

### 5. 📊 流派声学相似度与大对比
* **对比雷达图**：在 KMeans 弹窗中支持最多 4 个流派的 7 维声学特征重叠雷达对比。
* **极简声学相似度计算模型**：
  * 基于 7 维声学向量，通过两两流派对的**平均绝对误差 (MAE)** 距离倒数进行相似度计算：
    $$\text{Similarity} = \max(0, 1 - \text{MAE} \times 1.6) \times 100\%$$
  * 自动将相似度结果规约，并在简约的灰色标签条内展示，提供“高度相似 / 中度相似 / 差异显著”的分级判定。

### 6. 📋 多维属性编码单曲列表 (`SongTable`)
* **多层信息编码**：
  * 列表单曲左侧配备 **流派色圆点**。
  * 紧随其后配备 **KMeans 聚类色小圆点**，无需冗余文字即可传达歌曲流派与聚类双重属性。
  * 附带彩色流派小胶囊与热度百分比进度。
* **框选极速响应**：毫秒级响应散点图的 Brush 框选操作，过滤显示区域内的高热度歌曲。

## ⚙️ 本地部署与运行指南

### 🛠️ 环境依赖
* **Node.js** (推荐 `v16` 或 `v18+`)
* **npm**

### ⚛️ 启动前端服务
1. **进入项目目录**：
   ```bash
   cd music/spotify-vis
   ```
2. **安装依赖**：
   ```bash
   npm install
   ```
3. **启动开发服务器**：
   ```bash
   npm run dev
   ```
   启动后在浏览器打开终端提示的地址（默认 `http://localhost:5173/`）。

4. **构建生产版本**：
   ```bash
   npm run build
   ```

## 🗃️ 数据预处理与离线损耗优化
为了保证大屏可视化在复杂多维过滤、Voronoi 图实时渲染以及高频计算时的极端流畅度，原始 20MB 的 Spotify 歌曲数据集已采用 **Python 进行了离线预处理**：
1. **去噪与中文汉化**：通过定制的高防卫音乐字典，完成流派汉化、大类归纳及正则化过滤。
2. **高频歌词提取**：预先提取各流派的前 20 位歌词特征高频词，并对年代（1960-2020）均值进行整理，存入 `genre_details` 节点。
3. **降维与前端缓存**：保留核心声学数据直接内置于 `public/data.json`，在前端零开销、开箱即用。
